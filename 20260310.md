## Redes de Computadores: Modelo de Referência OSI

O modelo **OSI (Open Systems Interconnection)** é um framework conceitual de 7 camadas que padroniza as funções de um sistema de comunicação. Nesta aula, focamos nas camadas "inferiores", responsáveis pela transmissão física e entrega de dados entre dispositivos adjacentes.

Aqui está a tabela das 7 camadas do Modelo OSI, formatada em Markdown para o seu GitHub:

| Camada | Nome | Função Principal | Unidade (PDU) |
| --- | --- | --- | --- |
| **7** | **Aplicação** | Interação direta com o utilizador e software. | Dados |
| **6** | **Apresentação** | Tradução, compressão e criptografia. | Dados |
| **5** | **Sessão** | Estabelecimento e gestão de diálogos. | Dados |
| **4** | **Transporte** | Transferência ponto a ponto e correção de erros. | Segmentos |
| **3** | **Rede** | Endereçamento lógico e determinação de rotas. | Pacotes |
| **2** | **Enlace** | Endereçamento físico (MAC) e acesso ao meio. | Quadros (Frames) |
| **1** | **Física** | Transmissão de bits brutos pelo meio físico. | Bits |


---

### 1. Camada 1: Física (Physical Layer)

A camada física lida com a transmissão e recepção de **bits** brutos através de um meio físico. Ela define as especificações elétricas, mecânicas e funcionais.

* **Função:** Definir como os dados são convertidos em sinais (elétricos, luminosos ou rádio).
* **Equipamentos:** Hubs, repetidores, cabos (UTP, Fibra Óptica) e conectores (RJ45).
* **Exemplos de Protocolos/Padrões:** Ethernet (físico), Wi-Fi (físico), Bluetooth, USB.

---

### 2. Camada 2: Enlace de Dados (Data Link Layer)

A camada de enlace transforma a camada física em um link confiável. Ela organiza os bits em unidades chamadas **Quadros (Frames)**.

Esta camada é subdividida em duas:

1. **LLC (Logical Link Control):** Gerencia o controle de fluxo e erros.
2. **MAC (Media Access Control):** Controla o acesso ao meio físico e o endereçamento físico.

* **Endereçamento:** Utiliza o **Endereço MAC** (único e gravado no hardware).
* **Funções:** Detecção de erros, controle de fluxo e delimitação de quadros.
* **Equipamentos:** Switches e Pontes (Bridges).

---

### Comparação de Conceitos Chave

| Característica | Camada Física | Camada de Enlace |
| --- | --- | --- |
| **Unidade de Dados (PDU)** | Bits | Quadros (Frames) |
| **Identificação** | Nível de sinal (Volts/Luz) | Endereço MAC |
| **Foco principal** | Transmissão de sinais | Organização e controle de erros |
| **Hardware Comum** | Cabos e Hubs | Switches e Placas de Rede (NIC) |

---

> **Atenção:** Enquanto a Camada Física se preocupa se o "bit" chegou como 0 ou 1, a Camada de Enlace se preocupa em saber **quem** enviou esse conjunto de bits e se eles chegaram sem corrupção de dados.

## Diferenciando Dispositivos de Rede: Hub, Switch e Roteador

Para entender como uma rede funciona, precisamos saber em qual camada do Modelo OSI cada dispositivo opera e como eles tratam os dados.

### 1. Hub (Camada 1 - Física)

O Hub é um dispositivo "burro". Ele apenas recebe bits em uma porta e os replica para todas as outras.

* **Problema:** Gera muito tráfego desnecessário e colisões de dados.
* **Wi-Fi (O "Hub pelo Ar"):** O Wi-Fi funciona de forma semelhante a um Hub. Como o sinal de rádio é transmitido para todos os lados, todos os dispositivos no alcance "ouvem" o sinal, mas apenas o destinatário correto deve processá-lo.

### 2. Switch (Camada 2 - Enlace)

O Switch é inteligente. Ele aprende o **Endereço MAC** de cada dispositivo conectado.

* **Função:** Ele encaminha os **Quadros (Frames)** apenas para a porta onde o destino está conectado.
* **Vantagem:** Reduz o tráfego e aumenta a segurança da rede local.

### 3. Roteador (Camada 3 - Rede)

O Roteador conecta redes diferentes (ex: sua casa com a Internet).

* **Função:** Ele trabalha com **Endereços IP** e pacotes. É o responsável por decidir a melhor rota para os dados saírem de uma rede e chegarem a outra.
* **Contraste:** Enquanto o Switch olha para o "nome físico" (MAC), o roteador olha para o "endereço lógico" (IP).

### Laboratório Prático: Explorando IP e MAC no Windows

Abra o **Prompt de Comando (CMD)** e execute os passos abaixo para identificar os endereços da sua rede:

#### Passo 01: Identificar seu próprio endereço

Digite o comando abaixo para ver as configurações da sua placa de rede:

```bash
ipconfig

```

> **O que observar:** Anote o seu **Endereço IPv4** e o **Gateway Padrão** (que geralmente é o endereço IP do seu Roteador).

#### Passo 02: Descobrir o IP do "vizinho" (Dispositivos na rede)

Para ver quem mais está conectado na sua rede local, você pode usar ferramentas de varredura ou simplesmente tentar comunicar-se com outro IP que você saiba que existe.

#### Passo 03: Ver a tabela de endereços físicos (MAC)

Após haver comunicação na rede, o Windows armazena a relação entre IP e MAC. Use:

```bash
arp -a

```

* **Interface:** Exibe o seu IP.
* **Endereço Internet:** Mostra o IP do vizinho ou do roteador.
* **Endereço Físico:** Mostra o **MAC Address** (Camada 2) correspondente àquele IP.

---

> **Atenção:** Se o `arp -a` não mostrar o IP que você procura, tente dar um `ping [IP_DO_VIZINHO]` primeiro para forçar a atualização da tabela ARP.

## 🏛️ A História e a Estrutura do TCP/IP

### 1. Como Tudo Começou (Anos 60/70)

* **O Medo da Guerra:** No auge da Guerra Fria, os EUA precisavam de uma rede que não tivesse um "cérebro central". Se um nó fosse destruído por um ataque, a comunicação não poderia parar.
* **ARPANET:** O projeto nasceu na ARPA (hoje DARPA). Os "pais" da internet, **Vint Cerf e Bob Kahn**, desenvolveram o TCP/IP para permitir que redes diferentes se comunicassem.
* **Tolerância a Falhas:** Diferente do sistema de telefonia da época (onde se uma linha caía, a chamada caía), o TCP/IP foi feito para o **roteamento dinâmico**: o pacote de dados encontra outro caminho sozinho se um servidor sair do ar.

### 2. O Conceito de "Best Effort" (Melhor Esforço)

* A rede não garante que o caminho é perfeito, mas o protocolo **TCP** garante que, se um pedaço do dado sumir no caminho, ele será retransmitido. É por isso que o seu código chega inteiro no GitHub, mesmo passando por cabos submarinos instáveis.

---

### 3. TCP/IP vs. Modelo OSI (A Grande Diferença)

Muitas vezes os alunos confundem, mas a relação é de **Teoria vs. Prática**:

* **Modelo OSI (7 Camadas):** Criado pela ISO. É o modelo **teórico**, acadêmico, perfeito no papel para organizar como os protocolos *deveriam* funcionar.
* **Modelo TCP/IP (4 Camadas):** É o modelo **prático**. Ele "venceu" a guerra dos protocolos porque já estava funcionando e sendo usado enquanto o OSI ainda era discutido em reuniões.

**Na prática, o TCP/IP "esmaga" as camadas do OSI:**

1. As camadas de **Aplicação, Apresentação e Sessão** do OSI viram apenas **APLICAÇÃO** no TCP/IP.
2. A camada de **Transporte** é igual em ambos.
3. A camada de **Rede** do OSI vira **INTERNET** no TCP/IP.
4. As camadas de **Enlace e Física** do OSI viram **INTERFACE DE REDE** no TCP/IP.

### 🛡️ Relação com a Segurança (Habilidade 1.1)

* **Por que saber isso?** Para identificar ameaças, o aluno precisa saber onde elas atacam.
* Um ataque de **Engenharia Social** (Aula 01) acontece na camada de **Aplicação**.
* Um ataque de **DDoS** (Aula 02) pode acontecer na camada de **Transporte** ou **Internet**.
