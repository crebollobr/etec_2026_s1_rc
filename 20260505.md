
# 💡 Lab: Análise de Tráfego Real com Sockets e TCP

**Disciplina:** Redes de Computadores  
**Ambiente:** Servidor Remoto (via ttyd) + Wireshark Portable  
**Formato:** Individual ou Dupla  

---

## 1. Objetivo
O objetivo desta atividade é capturar e analisar o fluxo de comunicação entre um cliente (seu servidor na nuvem) e um servidor web externo, identificando a formação de **Sockets** (IP:Porta) e o processo de conexão **TCP (Handshake)**.

## 2. Ferramentas
- **Terminal Web (ttyd):** Acesso ao servidor Linux. https://ssh.chr.eti.br
- **tcpdump:** Ferramenta de linha de comando para captura de pacotes.
- **curl:** Ferramenta para realizar requisições HTTP.
- **Wireshark Portable:** Software para análise visual dos pacotes capturados.

---

## 3. Procedimento Experimental

### Passo 1: Escolha do Alvo e Resolução de Nome
Escolha um site da lista fornecida pelo professor. No terminal, descubra qual é o endereço IP desse servidor:
```bash
host nome-do-site.com
```
*Anote o endereço IP retornado.*

### Passo 2: Preparação da Captura (Sniffing)
Inicie o `tcpdump` para capturar apenas o tráfego que entra ou sai do IP que você descobriu. O arquivo será salvo diretamente na pasta de download do servidor.

**Comando:**
```bash
sudo tcpdump -i any host [IP_DO_SITE] -w /var/www/html/seu_nome.pcap
```
*Substitua `[IP_DO_SITE]` pelo número que você anotou e `seu_nome` pelo seu nome ou do grupo.*

### Passo 3: Gerando Tráfego
Abra uma **nova aba** no navegador com o terminal (ou aguarde o comando do professor) e execute uma requisição para o site:
```bash
curl -L http://nome-do-site.com
```
Após o texto do site aparecer na tela, volte ao terminal onde o `tcpdump` está rodando e pressione `CTRL + C` para encerrar a captura.

### Passo 4: Download e Análise
1. No navegador do notebook da ETEC, acesse: `https://curso.chr.eti.br/seu_nome.pcap`.
2. Abra o arquivo baixado usando o **Wireshark Portable**.
