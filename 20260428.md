# Plano de Aula: Fundamentos de Redes e Análise de Tráfego com Wireshark

## Parte 1: Base Tecnológica e Teoria

### 1. Tópicos de Redes de Computadores

#### Meios de Transmissão e Endereçamento Físico
O **Endereço MAC (Media Access Control)** é o identificador único gravado no hardware da placa de rede. Enquanto o IP serve para o roteamento global (chegar à rede de destino), o MAC é o responsável pela entrega final dentro da rede local (camada de enlace).

**Como verificar seu endereço físico:**
* **No Windows:** Abra o CMD e digite `ipconfig /all`. Procure por "Endereço Físico".
* **No Linux:** Abra o terminal e digite `ip add` ou `ifconfig`. Procure pelo campo `link/ether`.

#### Modelo TCP/IP
O modelo TCP/IP organiza as funções de rede em 4 camadas principais:

| Camada | Para que serve | Protocolos de Exemplo |
| :--- | :--- | :--- |
| **Aplicação** | Interface com o usuário e serviços de rede. | HTTP, HTTPS, DNS, SMTP |
| **Transporte** | Comunicação fim-a-fim, controle de fluxo e erro. | TCP, UDP |
| **Internet** | Endereçamento lógico e determinação da melhor rota. | IP (IPv4, IPv6), ICMP |
| **Acesso à Rede** | Transmissão física dos dados e endereçamento MAC. | Ethernet, Wi-Fi (802.11) |



---

### 2. Modelo Cliente/Servidor e Fluxo de Comunicação

Quando você acessa um site (ex: `https://site.com.br`), o seguinte fluxo ocorre:

1.  **DNS:** O computador consulta um servidor DNS para traduzir o nome em um **Endereço IP**.
2.  **Roteamento:** O sistema verifica se o IP de destino está na mesma rede. Caso não esteja, ele envia o pacote para o **Gateway** (roteador).
3.  **Encapsulamento:** O pacote é montado com:
    * **IP Origem:** Seu desktop | **IP Destino:** O servidor do site.
    * **MAC Origem:** Seu desktop | **MAC Destino:** O Gateway.
    * **Porta Origem:** Uma porta alta aleatória | **Porta Destino:** 443 (HTTPS).
4.  **NAT (Network Address Translation):** Como seu IP privado não navega na internet, o roteador substitui seu IP de origem pelo **IP Público** dele e troca a porta de origem por uma porta aleatória controlada por ele, criando uma tabela de mapeamento:

| IP Privado Desktop | Porta Origem | IP NAT (Público) | Porta NAT | IP Destino | Porta Destino |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 192.168.1.10 | 55030 | 200.150.10.5 | 60001 | 104.18.2.1 | 443 |

#### IPv4 vs IPv6
* **IPv4:** Esgotamento de endereços, exigindo o uso de NAT.
* **IPv6:** Todos os dispositivos podem ter IPs públicos. Um computador moderno geralmente possui:
    * **Localhost (`::1`):** A própria máquina.
    * **Link Local (`fe80::`):** Para comunicação dentro da mesma rede física.
    * **Global Unicast:** IP público para navegar na internet.

---

### 3. Protocolos de Transporte: TCP vs UDP

* **TCP (Transmission Control Protocol):** Orientado à conexão. Garante que os dados cheguem na ordem e sem erros através das flags **SYN** e **ACK** (Three-way Handshake).
    * *Uso:* Navegação web, e-mail, transferência de arquivos.
* **UDP (User Datagram Protocol):** Sem conexão. Envia os dados sem verificar se foram recebidos.
    * *Uso:* Streaming de vídeo/áudio, VoIP, jogos online (onde a velocidade é mais importante que a recuperação de perdas).

### Notas Adicionais sobre Endereços Físicos
O MAC é específico para redes Ethernet. Outros meios físicos possuem identificadores diferentes, como o **EUI-64** em algumas redes sem fio ou o identificador de circuito em conexões do tipo WAN ponto-a-ponto.

# Laboratório: Os Caminhos da Web (Investigando Conexões)

Neste laboratório, vocês vão agir como detetives de rede. Vamos descobrir com quem o seu computador está conversando, usando qual idioma (protocolo) e por qual porta.

### Passo a Passo da Investigação

1. **Descubra quem é você:** Abra o Prompt de Comando (CMD) e digite `ipconfig /all`. Anote o seu endereço IP atual.
2. **Escolha o Alvo:** Escolha um site para investigar. Dica: evite sites gigantes como o `www.google.com`, pois eles têm centenas de IPs e mudam a toda hora. Tente algo como `www.terra.com.br` ou o site da sua faculdade.
3. **Descubra o IP do Alvo:** No CMD, digite o comando `nslookup www.seusite.com.br`. Ele vai te devolver os endereços de rede daquele domínio. Anote os IPs que aparecerem.
4. **Navegue:** Abra o seu navegador de internet e acesse o site que você escolheu.
5. **O Flagrante (netstat):** Volte rapidamente para o CMD e digite o comando abaixo (substituindo pelo IP que você anotou). Use as aspas para o filtro funcionar corretamente:
   `netstat -na | find "IP_DO_SITE"`
6. **Teste de Persistência:** Feche a aba do site no navegador. Espere 5 minutos, vá no CMD e repita o comando `netstat`. A conexão ainda está lá ou sumiu?

### Relatório da Investigação (Responda):
Com base no resultado do seu `netstat`, preencha os dados abaixo:
* **Seu IP (Origem) e o IP do Site (Destino):**
* **Qual é a versão do IP usado na conexão?** (Dica: IPv4 tem o formato `192.168.X.X` e o IPv6 tem letras, números e dois pontos `2804:14c::...`).
* **Qual a porta do seu computador e qual a porta do site?**
* **A conexão é da web "antiga" ou da "nova"?** (Veja se você encontrou a sigla TCP ou UDP).

### A Grande Dúvida: Por que tem tráfego UDP na porta 443?

Se você abrir qualquer livro antigo de redes, ele dirá: *"Navegação web (HTTP/HTTPS) funciona na porta 80 e 443 usando exclusivamente o protocolo TCP."* Mas o aluno viu **UDP** indo para a porta **443**. O que é isso? O livro está errado? 

O livro está desatualizado. Vocês acabaram de flagrar a **Web Moderna (HTTP/3)** em ação! 

**A explicação:**
Tradicionalmente, a web usa o **TCP** porque ele é o "entregador com aviso de recebimento". Ele garante que os dados cheguem em ordem e sem perdas. O problema é que o TCP é "burocrático". Antes de enviar a página, ele faz o chamado *3-way handshake* (aperto de mãos), o que causa lentidão.

Para deixar a internet muito mais rápida, o Google criou um protocolo novo chamado **QUIC** (Quick UDP Internet Connections), que hoje é a base oficial do novo **HTTP/3**. 

O QUIC resolveu pegar carona no protocolo **UDP**. O UDP é o "entregador apressado": ele joga o pacote na sua casa e sai correndo, sem pedir assinatura. Ele é muito mais rápido, mas não garante a entrega. 

**O truque genial do QUIC:** Ele usa a velocidade pura do UDP na camada de transporte, mas recria toda a confiabilidade do TCP (checagem de erros e ordem dos pacotes) além de já embutir a criptografia (TLS 1.3) na própria conexão. 

**Resumo da Ópera:**
* Encontrou só **TCP na porta 443**? O navegador está usando a internet tradicional (HTTP/1.1 ou HTTP/2).
* Encontrou **UDP na porta 443**? O navegador e o servidor são modernos e estão usando o protocolo **QUIC (HTTP/3)**, garantindo a navegação mais rápida possível no mundo hoje.
