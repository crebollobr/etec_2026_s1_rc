## 📑 Parte 1: Base Teórica (35 min)

### 1. Firewall de rede × Firewall de aplicação (WAF)

| | Firewall comum (camada 3/4) | **WAF (camada 7 — aplicação)** |
|---|---|---|
| Olha para... | IP e **porta** | o **conteúdo** do pedido HTTP |
| Decisão tipo | "bloqueia a porta 23" | "bloqueia pedidos com `../etc/passwd`" |
| Onde fica | borda da rede | **na frente da aplicação web** (= seu proxy reverso!) |

O firewall comum não entende HTTP — para ele, todo tráfego na porta 443 é igual. O **WAF entende HTTP** e por isso consegue ver a diferença entre um pedido legítimo e um ataque. É por isso que o lugar natural do WAF é **exatamente onde está seu proxy reverso**: ele já vê todo pedido passar.

### 2. Os ataques que um WAF barra (camada de aplicação)

Todos viajam dentro de pedidos HTTP **aparentemente normais**:

- **Path traversal (`../`):** pedir `GET /../../etc/passwd` para escapar da pasta do site e ler arquivos do sistema.
- **SQL Injection:** enfiar `' OR 1=1 --` ou `UNION SELECT` num campo, para enganar o banco de dados.
- **XSS:** injetar `<script>` para rodar código no navegador de outra vítima.
- **Bots/scanners:** ferramentas como `sqlmap` e `nikto` se anunciam no cabeçalho `User-Agent` — um WAF reconhece e barra.
- **Flood / força bruta:** centenas de pedidos por segundo do mesmo IP — barrados por **limite de taxa (rate limit)**.

### 3. Duas filosofias de filtro

- **Lista de bloqueio (blocklist):** "deixa passar tudo, menos o que eu sei que é ruim." Fácil de começar, mas o atacante sempre inventa um padrão novo. É a que vamos usar hoje (didática e visível).
- **Lista de permissão (allowlist):** "bloqueia tudo, menos o que eu sei que é bom." Muito mais segura, mais trabalhosa. É o padrão profissional.

> 🧠 **Princípio de ouro da segurança:** *fail closed* — na dúvida, **bloqueie**. É melhor irritar um usuário legítimo do que deixar passar um ataque.

### 4. Onde o WAF se encaixa

```
   Atacante 😈            Usuário 🙂
        │                     │
        ▼                     ▼
   ┌───────────────────────────────┐
   │   WAF (seu proxy.py + filtros) │  ← inspeciona, decide: 403 ou repassa
   └───────────────────────────────┘
        │ só o tráfego limpo passa
        ▼
   ┌──────────┐
   │ mural.py │  (nunca vê o ataque)
   └──────────┘
```

### Etapa 2 — Transforme o proxy num WAF

Pegue seu `proxy.py` (aula 23/06) e crie a versão com **firewall**. A diferença é uma função `inspecionar()` que roda **antes** de encaminhar: se achar algo suspeito, devolve `403 Forbidden` e o pedido **nunca chega ao mural**.

```bash
cat > /root/waf.py << 'EOF'
from http.server import BaseHTTPRequestHandler, HTTPServer
import urllib.request, urllib.error, re, time
from urllib.parse import unquote

BACKEND = 'http://127.0.0.1:8000'

# --- assinaturas de ataque (blocklist) ---
REGRAS = [
    ('path traversal', re.compile(r'\.\./|/etc/passwd', re.I)),
    ('sql injection',  re.compile(r"(\bor\b\s+1\s*=\s*1|union\s+select|--|\bdrop\s+table\b)", re.I)),
    ('xss',            re.compile(r'<\s*script', re.I)),
]
FERRAMENTAS_ATAQUE = re.compile(r'sqlmap|nikto|nmap|masscan|havij', re.I)

# --- limite de taxa: no máximo LIMITE pedidos por JANELA segundos, por IP ---
LIMITE, JANELA = 15, 10
historico = {}   # ip -> lista de horários

def inspecionar(h):
    """Devolve o motivo do bloqueio, ou None se o pedido é limpo."""
    alvo = unquote(h.path)                       # decodifica %2e%2e etc.
    ua   = h.headers.get('User-Agent', '')

    # 1) corpo + caminho contra as assinaturas
    tam  = int(h.headers.get('Content-Length', 0))
    corpo = h.rfile.read(tam).decode('utf-8', 'ignore') if tam else ''
    h._corpo = corpo.encode()                    # guarda p/ reenviar ao backend
    alvo_total = alvo + ' ' + corpo
    for nome, rx in REGRAS:
        if rx.search(alvo_total):
            return nome

    # 2) ferramentas de ataque conhecidas
    if FERRAMENTAS_ATAQUE.search(ua):
        return 'ferramenta de ataque no User-Agent'

    # 3) limite de taxa por IP
    ip, agora = h.client_address[0], time.time()
    recentes = [t for t in historico.get(ip, []) if agora - t < JANELA]
    recentes.append(agora)
    historico[ip] = recentes
    if len(recentes) > LIMITE:
        return 'rate limit (muitos pedidos)'

    return None

class WAF(BaseHTTPRequestHandler):

    def tratar(self):
        motivo = inspecionar(self)
        if motivo:
            print(f'[BLOQUEADO 🚫] {self.command} {self.path}  -> {motivo}  (IP {self.client_address[0]})')
            corpo = f'{{"erro":"bloqueado pelo WAF","motivo":"{motivo}"}}'.encode()
            self.send_response(403)
            self.send_header('Content-Type', 'application/json; charset=utf-8')
            self.send_header('Content-Length', str(len(corpo)))
            self.end_headers()
            self.wfile.write(corpo)
            return

        print(f'[OK ✅] {self.command} {self.path}')
        req = urllib.request.Request(BACKEND + self.path,
                                     data=getattr(self, '_corpo', None) or None,
                                     method=self.command)
        if 'Content-Type' in self.headers:
            req.add_header('Content-Type', self.headers['Content-Type'])
        req.add_header('X-Forwarded-For', self.client_address[0])
        try:
            resp = urllib.request.urlopen(req)
            dados, codigo = resp.read(), resp.status
        except urllib.error.HTTPError as e:
            dados, codigo = e.read(), e.code
        self.send_response(codigo)
        self.send_header('Content-Type', 'application/json; charset=utf-8')
        self.send_header('Content-Length', str(len(dados)))
        self.end_headers()
        self.wfile.write(dados)

    do_GET = do_POST = do_PUT = do_DELETE = tratar

print('🛡️  WAF na porta 8080 protegendo o backend 8000')
HTTPServer(('0.0.0.0', 8080), WAF).serve_forever()
EOF
python3 /root/waf.py
```

### Etapa 3 — Tráfego legítimo passa

Terminal 2 — o usuário de bem usa o mural normalmente, pela porta do WAF (8080):

```bash
curl http://localhost:8080/
curl -X POST -d '{"texto":"recado normal de um usuario de bem"}' http://localhost:8080/
curl http://localhost:8080/
```

📝 **Anote:** no terminal do WAF aparece `[OK ✅]`. O pedido limpo passou direto.

### Etapa 4 — 😈 Vire atacante e ataque seu próprio firewall

Agora dispare os ataques da teoria **contra a porta 8080** e veja o WAF segurar cada um:

```bash
# 1) Path traversal — tentar ler arquivo do sistema
curl -i "http://localhost:8080/../../etc/passwd"

# 2) SQL injection no caminho
curl -i "http://localhost:8080/produtos?id=1' OR 1=1 --"

# 3) SQL injection no corpo de um POST
curl -i -X POST -d "{\"texto\":\"'; DROP TABLE recados; --\"}" http://localhost:8080/

# 4) XSS
curl -i "http://localhost:8080/busca?q=<script>alert(1)</script>"

# 5) Ferramenta de ataque se anunciando no User-Agent
curl -i -A "sqlmap/1.7" http://localhost:8080/

# 6) Flood — dispara 25 pedidos rápidos para estourar o rate limit
for i in $(seq 1 25); do curl -s -o /dev/null -w "%{http_code} " http://localhost:8080/; done; echo
```

📝 **Anote:**
- a) Qual **código de status** cada ataque recebeu (1 a 5)? Olhe o corpo da resposta — qual **motivo** o WAF informou?
- b) No item 6, a partir de qual pedido o status mudou de `200` para `403`? Por quê?
- c) Cole 5 linhas do **log do WAF** mostrando os `[BLOQUEADO 🚫]` com os motivos.
- d) **Prova de que o mural foi protegido:** olhe o terminal do mural (8000). Os ataques apareceram lá? (Não deveriam — o WAF barrou antes.)

### 🏅 Desafio extra (vale ponto) — Melhore o firewall

Escolha **pelo menos um**:

1. **Bloqueio por tamanho:** rejeite com `413` qualquer pedido cujo corpo passe de 1000 bytes (um truque comum de ataque é mandar payloads gigantes).
2. **Bloqueio de IP:** mantenha uma lista `IPS_BANIDOS` e devolva `403` na hora para quem estiver nela.
3. **Página de bloqueio amigável:** em vez do JSON, devolva um HTML bonito explicando que o pedido foi barrado (com `Content-Type: text/html`).
4. **Furar o próprio WAF:** ache **uma** forma de ataque que passe pelas regras atuais (dica: e se você codificar o `../` de outro jeito, ou usar `oR 1=1`?) e depois **conserte a regra** para pegá-la. Relate o ataque e o conserto — isso é o ciclo real de quem trabalha com segurança.
