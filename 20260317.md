# 2. Modelo OSI

O modelo OSI divide a comunicação em **7 camadas**.

| Camada           | Função                    |
| ---------------- | ------------------------- |
| 7 – Aplicação    | comunicação com o usuário |
| 6 – Apresentação | formatação e codificação  |
| 5 – Sessão       | controle de sessões       |
| 4 – Transporte   | controle de envio         |
| 3 – Rede         | endereçamento IP          |
| 2 – Enlace       | comunicação na rede local |
| 1 – Física       | transmissão de bits       |

Exemplo ao acessar um site:

* Aplicação → HTTP
* Transporte → TCP
* Rede → IP
* Enlace → Ethernet
* Física → cabos ou Wi-Fi

---

# 3. Modelo TCP/IP

Na prática da internet usamos o modelo **TCP/IP**, que possui menos camadas.

| Camada TCP/IP | Correspondência OSI |
| ------------- | ------------------- |
| Aplicação     | 5,6,7               |
| Transporte    | 4                   |
| Internet      | 3                   |
| Acesso à rede | 1,2                 |

Protocolos importantes:

* HTTP
* TCP
* IP
* ICMP

---

# 4. Protocolo ICMP

ICMP é usado para **diagnóstico de rede**.

Exemplo mais conhecido:

```
ping
```

Quando executamos:

```
ping servidor
```

ocorre:

1. envio de **ICMP Echo Request**
2. resposta **ICMP Echo Reply**

Isso permite verificar:

* se o host está ativo
* tempo de resposta
* perda de pacotes

---

# 5. Protocolo HTTP

HTTP é o protocolo usado para **transferência de páginas web**.

Exemplo:

Quando acessamos:

```
http://servidor
```

o navegador envia:

```
HTTP Request
```

O servidor responde:

```
HTTP Response
```

Conteúdo retornado:

* HTML
* CSS
* imagens
* scripts

# 6. Simulação no Packet Tracer

Vamos simular duas redes simples usando o Cisco Packet Tracer.

---

# 7. Cenário 1 – Computador → Servidor

Topologia:

```
PC -------- Server
```

Passos:

1. adicionar **PC**
2. adicionar **Server**
3. conectar com **cabo Ethernet**
4. configurar IP manualmente

Exemplo:

PC

```
192.168.1.10
```

Server

```
192.168.1.20
```

---

# Teste de conectividade

No PC executar:

```
ping 192.168.1.20
```

Observar:

* pacotes ICMP
* resposta do servidor

---

# Visualização no modo Simulation

No Packet Tracer:

1. mudar para **Simulation Mode**
2. executar **ping**
3. observar o pacote atravessando a rede

O simulador mostrará:

* camada física
* camada de enlace
* camada IP
* camada ICMP

---

# 8. Cenário 2 – Computador → Roteador → Servidor

Topologia:

```
PC ---- Router ---- Server
```

Passos:

1. adicionar **Router**
2. conectar os dispositivos
3. configurar IP nas interfaces do roteador

Exemplo:

PC

```
192.168.1.10
```

Router

```
192.168.1.1
```

Server

```
192.168.2.10
```

---

# Configurar Gateway

No PC:

```
gateway: 192.168.1.1
```

No servidor:

```
gateway: 192.168.2.1
```

---

# Teste com ping

Executar:

```
ping 192.168.2.10
```

Agora o pacote fará o caminho:

```
PC → Router → Server
```

---

# 9. Observando o percurso do pacote

No modo **Simulation** podemos observar:

1. criação do pacote
2. envio pela rede
3. passagem pelo roteador
4. chegada ao servidor
5. resposta retornando

Camadas envolvidas:

Aplicação
Transporte
Internet
Enlace
Física

---

# 10. Teste HTTP

No Packet Tracer:

1. abrir **Web Browser no PC**
2. acessar o IP do servidor

```
http://192.168.2.10
```

O simulador mostrará:

* requisição HTTP
* resposta do servidor
