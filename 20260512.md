

# Roteiro de Laboratório: Proxy e Proxy Reverso na Prática

Neste laboratório, você vai aprender como manipular o tráfego de rede usando proxies. Como estamos em um ambiente com restrições de administrador, usaremos técnicas de "Living off the Land" (usar ferramentas nativas como o Python) para contornar os bloqueios e montar nossa infraestrutura.

## 1. Teoria Rápida: Quem é quem?

* **Proxy (Forward Proxy):** É o **seu** intermediário. Você pede a ele para acessar um site, ele vai até a internet, baixa e te entrega. É usado para filtrar acessos (como o da escola) ou para contornar bloqueios.
* **Proxy Reverso (Reverse Proxy):** É o intermediário do **servidor**. Você acessa o Proxy Reverso achando que é o site oficial. Ele, em silêncio, busca o conteúdo num servidor interno (escondido) e te entrega. É usado para segurança e balanceamento de carga.

## 2. Prática 1: Furando o bloqueio com Forward Proxy em Python

Como o laboratório bloqueia arquivos `.exe` desconhecidos de proxy, vamos construir o nosso próprio proxy usando um script em Python. Ele vai receber nossa navegação e repassar para o proxy da escola.

**Passo A: Criando o seu Proxy**

1. Na sua Área de Trabalho, crie uma pasta chamada **`MeuProxy`**.
2. Abra o **Bloco de Notas**.
3. Copie e cole **exatamente** o código abaixo:

```python
import http.server
import urllib.request

# Configurações do Proxy
PORTA_LOCAL = 8080
PROXY_ESCOLA = "http://172.16.0.2:3128" # Altere se o professor indicar outro IP

class Proxy(http.server.SimpleHTTPRequestHandler):
    def do_GET(self):
        # Ensina o Python a passar pelo proxy da escola
        proxy_support = urllib.request.ProxyHandler({'http': PROXY_ESCOLA})
        opener = urllib.request.build_opener(proxy_support)
        urllib.request.install_opener(opener)
        
        # Busca o site e devolve para você
        self.send_response(200)
        self.end_headers()
        try:
            self.wfile.write(urllib.request.urlopen(self.path).read())
        except Exception as e:
            self.wfile.write(bytes(str(e), 'utf-8'))

print(f"Proxy local rodando na porta {PORTA_LOCAL}...")
http.server.HTTPServer(('127.0.0.1', PORTA_LOCAL), Proxy).serve_forever()

```

4. Vá em **Arquivo > Salvar Como**.
5. Salve dentro da pasta `MeuProxy` com o nome **`proxy_local.py`** (certifique-se de que o "Tipo" esteja como "Todos os Arquivos" para não salvar como .txt).

**Passo B: Executando e Testando**

1. Abra o **CMD** (Prompt de Comando) dentro da pasta `MeuProxy` (clique na barra de endereços da pasta, digite `cmd` e dê Enter).
2. Execute o seu script:
```cmd
python proxy_local.py

```


*(A tela vai mostrar "Proxy local rodando na porta 8080...". Deixe essa janela aberta!)*
3. Abra as **Configurações de Proxy** do Windows (ou do navegador).
4. Ative o uso de proxy manual e coloque: **Endereço:** `127.0.0.1` e **Porta:** `8080`.
5. Tente acessar um site `http` no navegador. Seu tráfego agora está passando pelo seu script Python, que fura o bloqueio através do proxy da escola!

## 3. Prática 2: Mascarando um servidor com Proxy Reverso

Agora vamos criar um cenário corporativo: um site em Python rodando em uma porta "escondida" (9000), e o **Nginx** na porta oficial (80) fazendo o papel de segurança e repassando os acessos.

**Passo A: O Site Escondido (Back-end)**

1. Crie uma pasta na Área de Trabalho chamada **`SiteSecreto`**.
2. Dentro dela, crie um arquivo chamado **`index.html`** e escreva dentro: *"Acesso liberado via Proxy Reverso!"*.
3. Abra um **novo CMD** dentro da pasta `SiteSecreto`.
4. Suba o servidor Python na porta 9000:
```cmd
python -m http.server 9000

```


*(Deixe essa janela aberta).*

**Passo B: Baixando o Nginx (O Portão de Entrada)**

1. Acesse o site oficial de download: 👉 `https://nginx.org/en/download.html`
2. Na seção *Mainline version*, clique na versão para Windows (ex: **`nginx/Windows-1.x.x`**) para baixar o `.zip`.
3. Extraia o arquivo `.zip` baixado na sua Área de Trabalho.

**Passo C: Configurando a regra do Nginx**

1. Entre na pasta do Nginx que você extraiu, e depois na pasta **`conf`**.
2. Encontre o arquivo **`nginx.conf`**, clique com o botão direito e abra com o **Bloco de Notas**.
3. **Apague absolutamente tudo** e cole esta configuração mínima:

```nginx
events {}

http {
    server {
        listen 80;
        
        location / {
            proxy_pass http://127.0.0.1:9000;
        }
    }
}

```

4. Salve (`Ctrl + S`) e feche o Bloco de Notas.

**Passo D: A Mágica do Proxy Reverso**

1. Volte para a pasta principal do Nginx (onde está o `nginx.exe`).
2. Abra um **novo CMD** nessa pasta e digite:
```cmd
nginx.exe

```


*(A tela vai piscar. O Nginx agora está rodando em segundo plano).*
3. **O Teste:** Vá no navegador e digite apenas a porta padrão:
```text
http://localhost



1. No CMD do Nginx, digite: `nginx.exe -s quit` para desligá-lo.
2. Feche a janela do Python (porta 9000).
3. Feche a janela do seu script Proxy Local e desative o proxy nas configurações do Windows.
