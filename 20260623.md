## 📑 Parte 1: Base Teórica (35 min)

### 1. Forward proxy × Reverse proxy (relembrando 12/06)

Na aula de proxy (12/06) você viu o **forward proxy**: o intermediário **do cliente** (o proxy da escola que filtra seu acesso). Hoje é o oposto:

| | Forward Proxy | **Reverse Proxy** |
|---|---|---|
| Está do lado de... | quem **navega** (cliente) | quem **serve** (servidor) |
| Protege/serve... | o usuário | a aplicação |
| O cliente sabe que existe? | sim, ele configurou | **não**, acha que fala direto com o site |
| Exemplo | proxy da escola | Nginx na frente do seu mural |

### 2. Por que pôr um proxy reverso na frente?

Um único porteiro na entrada resolve vários problemas de uma vez:

- 🔐 **Termina o TLS:** o proxy cuida do HTTPS/certificado; a aplicação atrás fala HTTP simples na rede interna. Um lugar só para gerenciar certificado.
- 🙈 **Esconde o backend:** o atacante vê o Nginx, não a porta 8000 da sua aplicação nem sua estrutura interna.
- ⚖️ **Balanceia carga:** com 3 cópias da aplicação, o proxy distribui os pedidos entre elas.
- 🛡️ **Ponto único de controle:** log, limite de taxa, bloqueio, cache — tudo num lugar só (a deixa do firewall da próxima aula).

### 3. O detalhe que todo mundo erra: os cabeçalhos

Quando o proxy repassa o pedido, o backend passa a ver o **IP do proxy** (127.0.0.1), não o do cliente real. Para não perder essa informação, o proxy injeta cabeçalhos:

- `Host` — qual domínio o cliente pediu;
- `X-Real-IP` / `X-Forwarded-For` — o IP verdadeiro do cliente.

Sem isso, o log da aplicação registra "todo mundo veio de 127.0.0.1" — inútil. Guarde esse `X-Forwarded-For`: ele é a base da contagem por IP no firewall da próxima aula.

### 4. O desenho de hoje

```
   Cliente (curl/navegador)
        │  HTTPS :8443
        ▼
   ┌──────────────┐   HTTP :8000   ┌─────────────┐
   │ Proxy reverso │ ─────────────▶ │  mural.py    │
   │ (Nginx/Python)│ ◀───────────── │  (backend)   │
   └──────────────┘    resposta     └─────────────┘
        ▲ termina o TLS, injeta cabeçalhos, registra log
```

### Etapa 2 — Nginx como proxy reverso com HTTPS

Use o certificado que você fabricou na aula passada (`/root/pki/servidor.crt` e `servidor.key`). No **terminal 2**, crie a configuração:

```bash
cat > /etc/nginx/sites-available/mural.conf << 'EOF'
server {
    listen 8443 ssl;
    server_name localhost;

    ssl_certificate     /root/pki/servidor.crt;
    ssl_certificate_key /root/pki/servidor.key;

    location / {
        proxy_pass http://127.0.0.1:8000;       # encaminha para o mural
        proxy_set_header Host $host;            # preserva o domínio pedido
        proxy_set_header X-Real-IP $remote_addr; # preserva o IP do cliente
    }
}
EOF
ln -sf /etc/nginx/sites-available/mural.conf /etc/nginx/sites-enabled/mural.conf
nginx -t          # testa a sintaxe ANTES de subir (faça sempre!)
nginx -s reload 2>/dev/null || nginx
```

Teste o caminho completo cliente → Nginx (HTTPS) → mural (HTTP):

```bash
curl --cacert /root/pki/ca_raiz.crt https://localhost:8443/
curl --cacert /root/pki/ca_raiz.crt -X POST -d '{"texto":"via proxy reverso"}' https://localhost:8443/
curl --cacert /root/pki/ca_raiz.crt https://localhost:8443/
```

📝 **Anote:** você acessou na porta **8443 com HTTPS**, mas a resposta veio do mural que só fala **HTTP na 8000**. Quem fez a "tradução" de HTTPS para HTTP? Em que porta o mural está, e o cliente precisou saber disso?

> 💻 **No Windows:** é o mesmíssimo bloco `server { listen 8443 ssl; ... proxy_pass ... }` do `nginx.conf` que vocês montaram em 02/06 — só trocando o backend Node pelo seu mural Python. Lembre: `nginx.exe -s reload` para aplicar, nunca abrir o `nginx.exe` duas vezes.

Pare o Nginx para o próximo passo: `nginx -s stop`

### Etapa 3 — Agora ESCREVA seu próprio proxy reverso em Python

O Nginx é uma caixa-preta. Vamos abrir essa caixa: um proxy reverso cabe em ~30 linhas. Ele recebe o pedido, **repassa** ao backend, e devolve a resposta — registrando tudo no caminho.

```bash
cat > /root/proxy.py << 'EOF'
from http.server import BaseHTTPRequestHandler, HTTPServer
import urllib.request, urllib.error

BACKEND = 'http://127.0.0.1:8000'   # para onde encaminhar

class ProxyReverso(BaseHTTPRequestHandler):

    def encaminhar(self):
        destino = BACKEND + self.path
        tam = int(self.headers.get('Content-Length', 0))
        corpo = self.rfile.read(tam) if tam else None

        # LOG: o proxy vê tudo que passa
        print(f'[PROXY] {self.command} {self.path}  (cliente {self.client_address[0]})')

        req = urllib.request.Request(destino, data=corpo, method=self.command)
        if 'Content-Type' in self.headers:
            req.add_header('Content-Type', self.headers['Content-Type'])
        # injeta o IP real do cliente para o backend (como o Nginx faz)
        req.add_header('X-Forwarded-For', self.client_address[0])

        try:
            resp = urllib.request.urlopen(req)
            dados, codigo = resp.read(), resp.status
            ctype = resp.headers.get('Content-Type', 'text/plain')
        except urllib.error.HTTPError as e:
            dados, codigo = e.read(), e.code            # repassa o erro do backend
            ctype = 'application/json'

        self.send_response(codigo)
        self.send_header('Content-Type', ctype)
        self.send_header('Content-Length', str(len(dados)))
        self.end_headers()
        self.wfile.write(dados)

    # todos os verbos passam pelo mesmo encaminhamento
    do_GET = do_POST = do_PUT = do_DELETE = encaminhar

print('Proxy reverso Python na porta 8080 -> backend 8000')
HTTPServer(('0.0.0.0', 8080), ProxyReverso).serve_forever()
EOF
python3 /root/proxy.py
```

(O mural da etapa 1 continua rodando no terminal 1.) No **terminal 2**, bata no **proxy** (8080), não no mural:

```bash
curl http://localhost:8080/
curl -X POST -d '{"texto":"passei pelo proxy Python"}' http://localhost:8080/
curl http://localhost:8080/
curl -X DELETE http://localhost:8080/1
```

📝 **Anote:** olhe o **terminal do proxy** — ele registrou cada pedido (`[PROXY] GET /`, `[PROXY] POST /`...). O cliente falou com a porta **8080**, mas quem respondeu foi o mural na **8000**. Cole 3 linhas do log do proxy.
