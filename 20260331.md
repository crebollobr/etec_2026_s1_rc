

## 📒 Notebook: Explorando Protocolos de Rede (HTTP & SMTP)

Os protocolos **HTTP** (Web) e **SMTP** (E-mail) são a base da comunicação na Internet. Ambos são baseados em **texto plano**, o que nos permite "conversar" com os servidores usando ferramentas simples.

### 1. HTTP via Telnet (Porta 80 - Sem Criptografia)
O HTTP (HyperText Transfer Protocol) funciona no modelo requisição-resposta. Na porta 80, os dados viajam abertos.

**Comandos:**
1. Abra o terminal e conecte-se:
   `telnet google.com 80`
2. Após conectar (pode não aparecer nada, apenas o cursor), digite exatamente:
   `GET / HTTP/1.1`
   `Host: google.com`
3. **Aperte ENTER duas vezes** (obrigatório para sinalizar o fim da requisição).

> **O que acontece:** O servidor retorna o código `200 OK` (ou um redirecionamento `301`) e o código HTML da página.

---

### 2. HTTP via OpenSSL (Porta 443 - Com Criptografia/HTTPS)
Hoje, a maioria dos sites exige HTTPS. O Telnet não consegue "falar" TLS/SSL, por isso usamos o `openssl`.

**Comandos:**
1. Conecte-se com segurança:
   `openssl s_client -connect google.com:443`
2. Uma tonelada de certificados aparecerá. Quando estabilizar, digite:
   `GET / HTTP/1.1`
   `Host: google.com`
3. **Aperte ENTER duas vezes.**



---

### 3. SMTP: Enviando E-mail Manualmente
Antes de conectar ao servidor de e-mail, precisamos saber quem cuida das mensagens daquele domínio usando o comando `dig`.

#### Passo A: Descobrir o servidor (MX Record)
`dig sapo.pt MX +short`
*O comando retornará algo como `10 mx.sapo.pt`. Use esse endereço no passo seguinte.*

#### Passo B: A Conversa SMTP (RFC 5321/5322)
Agora, vamos conectar na porta 25 do servidor encontrado.

**Sequência de Comandos:**
1.  `telnet [endereço-mx] 25`
2.  `EHLO seu-nome` (Apresentação)
3.  `MAIL FROM:<seu-email@dominio.com>` (Remetente)
4.  `RCPT TO:<destino@dominio.com>` (Destinatário)
5.  `DATA` (Inicia o corpo da mensagem)
6.  **Cabeçalhos (Obrigatórios para não cair no Spam):**
    ```text
    Subject: Teste de Protocolo
    From: <seu-email@dominio.com>
    To: <destino@dominio.com>
    Message-ID: <12345@dominio.com>
    Date: Wed, 25 Mar 2026 00:00:00 -0300

    Aqui vai o texto da mensagem.
    ```
7.  `.` (Um ponto final sozinho em uma linha para enviar)
8.  `QUIT` (Para fechar a conexão)

### Resumo dos Protocolos

| Protocolo | Porta Padrão | Função | Comando de Teste |
| :--- | :--- | :--- | :--- |
| **HTTP** | 80 | Transferência de hipertexto (Web) | `telnet` |
| **HTTPS** | 443 | HTTP sobre camada segura (TLS) | `openssl s_client` |
| **SMTP** | 25 / 587 | Envio de correio eletrônico | `telnet` |
| **DNS (MX)** | 53 | Resolução de servidores de e-mail | `dig` |
