# 🌐 Aula: Fundamentos de Redes, Modelo TCP/IP e Análise com Wireshark

Este repositório contém o material de apoio para a aula sobre tópicos fundamentais de redes de computadores, protocolos da internet e laboratório prático de análise de tráfego.

## 📖 Parte 1: Base Tecnológica e Teórica 

Nesta primeira etapa, abordaremos os conceitos estruturais de como os computadores se comunicam em rede, divididos pelos seguintes temas:

### 1. Tópicos de Redes de Computadores
* **Meios de Transmissão e Adaptadores:** Como a informação viaja e o papel das Placas de Interface de Rede (NICs).
* **Endereçamento Físico:** A identidade única de hardware (MAC Address) de cada adaptador na rede local.
* **Modelo Cliente/Servidor:** A arquitetura padrão da web, onde clientes (ex: seu navegador) fazem requisições e servidores entregam a resposta.

### 2. O Modelo TCP/IP e Suas Camadas
O Modelo TCP/IP é a espinha dorsal da Internet. Vamos detalhar as camadas que serão vistas no laboratório:

* **Camada de Acesso à Rede (Enlace):**
    * *Função:* Lida com a transmissão física e o endereçamento local via endereços MAC.
    * *Protocolo visto no lab:* Ethernet II.
* **Camada de Internet:**
    * *Função:* Roteamento de pacotes e endereçamento lógico.
    * *Protocolo visto no lab:* IPv4 (Endereços IP).
* **Camada de Transporte:**
    * *Função:* Garante a entrega dos dados e lida com portas lógicas.
    * *Protocolo visto no lab:* **TCP**, que garante a entrega ordenada e confiável.
* **Camada de Aplicação:**
    * *Função:* Onde os dados são consumidos pelos programas do usuário.
    * *Protocolo visto no lab:* **HTTP** (navegação web sem criptografia).

## 🔬 Parte 2: Laboratório Prático com Wireshark 

Nesta etapa, usaremos o **Wireshark** para abrir um "dump" (uma gravação) de tráfego real. 

### 📥 Preparação e Download do Dump
Nós usaremos uma captura oficial de exemplo disponibilizada pela própria fundação do Wireshark.
1. Baixe o arquivo de dump clicando neste link: [**http.cap**](https://wiki.wireshark.org/SampleCaptures?action=AttachFile&do=get&target=http.cap)
2. Abra o arquivo `http.cap` utilizando o Wireshark.

### 📝 Atividades Práticas
Com o arquivo aberto, explore as camadas do pacote para responder às 5 questões abaixo:

**1. Analisando as Conversas TCP (Camada de Transporte)**
* **Ação:** No menu superior, clique em **Statistics** (Estatísticas) > **Conversations** (Conversações). Vá até a aba **TCP**.
* **Pergunta:** Olhando para a tabela, **quantas conversas TCP** distintas ocorreram nessa captura de tráfego? 

**2. Identificando Protocolos Web (Camada de Aplicação)**
* **Ação:** Feche a janela de estatísticas. Na barra de filtros principal, digite `http` e pressione Enter. Clique no primeiro pacote listado (que tem um "Info" indicando uma requisição).
* **Pergunta:** Expandindo a camada **"Hypertext Transfer Protocol"** no painel do meio, qual foi o método de requisição utilizado (ex: GET, POST) e qual arquivo o cliente estava tentando baixar (URI)?

**3. Identificando as Portas (Camada de Transporte)**
* **Ação:** Com o mesmo pacote HTTP da atividade anterior selecionado, recolha a camada HTTP e expanda a camada **"Transmission Control Protocol" (TCP)**.
* **Pergunta:** Qual é a **Porta de Origem** (Source Port - porta do cliente) e a **Porta de Destino** (Destination Port - porta do servidor web)?

**4. Mapeando o Roteamento Lógico (Camada de Internet)**
* **Ação:** Limpe o filtro `http` clicando no "X" da barra de buscas. Selecione o pacote número 4 da lista. Expanda a camada **"Internet Protocol Version 4"**.
* **Pergunta:** O modelo de rede exige endereços lógicos. Quais são os endereços **IP de Origem** (Source Address) e **IP de Destino** (Destination Address) registrados neste pacote?

**5. Descobrindo os Endereços Físicos (Camada de Enlace)**
* **Ação:** Com o pacote número 4 ainda selecionado, expanda a primeira camada principal do pacote: **"Ethernet II"**.
* **Pergunta:** Quais são os endereços **MAC** (físicos) de origem e destino registrados na placa de rede para este pacote?
