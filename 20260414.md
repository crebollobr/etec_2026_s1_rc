## 💻 Laboratório Prático: Reconhecimento e Roteamento de Redes no Linux

**Objetivo:** Investigar a configuração de rede local do seu equipamento, entender como os pacotes viajam até a internet e diagnosticar a conectividade usando ferramentas nativas do Linux.

### 🕵️‍♂️ Missão 1: Reconhecimento do Terreno Local
Antes de acessar a internet, você precisa entender onde o seu computador está na rede. Use o terminal do Linux para descobrir as informações abaixo e anote os comandos utilizados.

1. **Qual é o endereço IP e a máscara de rede do seu computador?**
   * *Dica:* Procure por comandos que exibam as interfaces de rede e seus endereços.
2. **Qual é a rota default (gateway)?**
   * *Dica:* Qual comando mostra a tabela de roteamento do sistema operacional? O gateway é o seu "porta-voz" para a internet.
3. **Qual é o endereço MAC (físico) do gateway?**
   * *Dica:* O seu computador precisa traduzir o IP do gateway para um endereço físico da rede local. Qual protocolo (e qual comando) exibe a tabela que faz esse mapeamento? *(Nota: se não aparecer de imediato, tente enviar um ping para o IP do gateway primeiro).*

### 🛠️ Missão 3: Testando a Comunicação
Agora vamos verificar se a teoria está funcionando na prática.

1. **Ping restrito:** Faça um teste de conectividade (ping) para o IP do Google DNS (`8.8.8.8`). O detalhe é que você deve enviar **exatamente 4 pacotes** e fazer o comando parar sozinho.
   * *Tarefa:* Cole o comando utilizado e a resposta do terminal aqui. Pesquise no manual (`man ping`) qual parâmetro limita a contagem de pacotes no Linux.
2. **Resolução de Nomes:** Como não decoramos IPs, usamos DNS. Verifique as informações do domínio `google.com` usando a ferramenta `dig` (não utilize o `nslookup`).
   * *Tarefa:* Olhando o resultado do comando, qual é o endereço IP (ou endereços) que o servidor DNS retornou para o google.com? 
3. **Mapeamento de Serviços:** O que está rodando na sua máquina neste momento?
   * *Tarefa:* Descubra quais serviços estão escutando na rede local e em quais portas. Pesquise por comandos que mostram "sockets" estatísticos ou o status da rede.
   * *Pesquisa:* Escolha duas portas que apareceram abertas no seu resultado e pesquise na internet qual serviço geralmente utiliza essas portas.
