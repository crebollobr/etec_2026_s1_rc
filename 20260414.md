# 🌐 Aula: Fundamentos de Redes, Modelo TCP/IP e Análise com Wireshark

Este repositório contém o material de apoio para a aula sobre tópicos fundamentais de redes de computadores, protocolos da internet e laboratório prático de análise de tráfego.

## 📖 Parte 1: Base Tecnológica e Teórica 

Nesta etapa, abordaremos como os computadores se comunicam em rede:

### 1. Tópicos de Redes de Computadores
* **Meios de Transmissão:** Como a informação viaja e o papel das Placas de Rede (NICs).
* **Endereçamento:** A identidade física de hardware (MAC Address) na rede local.
* **Modelo Cliente/Servidor:** A arquitetura padrão da web (quem pede os dados e quem entrega).

### 2. O Modelo TCP/IP e Suas Camadas
* **Camada de Enlace (Acesso à Rede):** Lida com a transmissão física e o endereçamento local (MAC). *Protocolo prático: Ethernet.*
* **Camada de Internet:** Responsável pelo roteamento e endereçamento lógico. *Protocolo prático: IP (IPv4).*
* **Camada de Transporte:** Garante a entrega dos dados e lida com portas lógicas. *Protocolos práticos: TCP e UDP.*
* **Camada de Aplicação:** Onde os dados são gerados pelo usuário. *Protocolos práticos: HTTP, DNS.*

## 🔬 Parte 2: Laboratório Prático com Wireshark 

Nesta atividade, você usará o **Wireshark** para investigar um "dump" (captura) de tráfego de rede contendo centenas de pacotes e vários protocolos diferentes operando ao mesmo tempo. Seu objetivo é usar filtros para responder às missões.

### 📥 Passo 1: Download do Arquivo (Link Direto)
Baixe o arquivo de captura de tráfego (.pcap) clicando no link direto abaixo (basta salvar e abrir no Wireshark):
👉 [**Download do Dump de Rede (test.pcap)**](https://github.com/appneta/tcpreplay/raw/master/test/test.pcap)

### 📝 Passo 2: Missões de Análise de Tráfego

Abra o arquivo baixado no Wireshark e utilize os menus e a barra de filtros para responder de forma clara às 5 perguntas abaixo:

**1. Mapeando o terreno (Quais protocolos existem aqui?)**
* **Ação:** No menu superior do Wireshark, vá em **Statistics** (Estatísticas) > **Protocol Hierarchy** (Hierarquia de Protocolos).
* **Pergunta:** Olhando a árvore gerada, liste pelo menos 3 protocolos da Camada de Aplicação (final da árvore) que estão presentes neste arquivo de captura.

**2. Quantas conversas TCP existem?**
* **Ação:** No menu superior, vá em **Statistics** > **Conversations** (Conversações) e clique na aba **TCP**.
* **Pergunta:** Olhando para a tabela de resultados, qual é o número exato de conversas (conexões) TCP diferentes que ocorreram neste dump?

**3. Identificando tráfego Web (Existe protocolo HTTP?)**
* **Ação:** Feche as estatísticas. Na barra principal de filtros (no topo), digite `http` e dê Enter.
* **Pergunta:** Existem pacotes HTTP nessa captura? Se sim, clique em um pacote com a informação "GET" e identifique: qual foi o site (Host) que o usuário tentou acessar?

**4. Resolvendo Nomes (Filtrando o DNS)**
* **Ação:** Limpe o filtro anterior, digite `dns` e dê Enter. Clique no primeiro pacote cuja coluna "Info" tenha a palavra "query". Expanda a linha *Domain Name System* nos detalhes do pacote.
* **Pergunta:** Qual foi o domínio (nome do site) que a máquina estava tentando descobrir o endereço IP? 

**5. Quem é o maior comunicador da rede?**
* **Ação:** Vá novamente em **Statistics** > **Endpoints** (Pontos de Extremidade) e clique na aba **IPv4**.
* **Pergunta:** Clique na coluna *Bytes* para ordenar do maior para o menor. Qual é o Endereço IP que enviou/recebeu a maior quantidade de dados nessa rede?
